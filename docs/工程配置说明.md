# 工程配置（单一来源）

## 原则

- **唯一维护源**：`project.yml`（XcodeGen 规格）。
- **应用显示名**：`INFOPLIST_KEY_CFBundleDisplayName`（当前为 **看盘啦**），与 `README` 中的产品名一致；改名称时只改 `project.yml` 后 **generate**，勿与 `project.pbxproj` 双头维护。
- **生成物**：`ZiWeiDoushuDianLiangXingKong.xcodeproj/project.pbxproj` 由 XcodeGen 根据 `project.yml` **生成**，不作为长期手改对象。
- **不要双头维护**：避免同时在 Xcode 里改 `project.pbxproj`、又在另一处改 `project.yml` 而不同步。若应急在 Xcode 中改过工程文件，请把变更**合并回 `project.yml`**，再执行下方命令覆盖生成。

## 修改流程

1. 编辑根目录下的 `project.yml`（增删 Target、改 Bundle ID、部署版本、编译选项等）。
2. 在 **macOS** 上安装 [XcodeGen](https://github.com/yonaskolb/XcodeGen)（例如 `brew install xcodegen`）。
3. 在仓库根目录执行（与下面脚本等价）：

```bash
xcodegen generate --spec project.yml
```

也可在 macOS 上执行仓库内脚本（需已安装 XcodeGen）：

```bash
chmod +x scripts/regenerate-xcode-project.sh
./scripts/regenerate-xcode-project.sh
```

4. 用 Xcode 打开工程，确认可编译、测试（⌘U）。

## 与仓库中 pbxproj 的关系

仓库里仍提交 `project.pbxproj`，是为了克隆后**未装 XcodeGen** 时也能直接双击打开工程。权威配置仍是 **`project.yml`**；若两者不一致，以 **`project.yml` + 重新 generate** 为准。
