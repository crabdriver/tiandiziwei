# Wave 3 APK 样本工作笔记

本笔记先冻结第三批样本的 APK 侧证据，再据此编写 JSON fixture。以下内容仅记录真实 APK 运行时观察到的输入串、截图与人工读取结果，不引用 iOS 输出反填。

## 样本 A: 主星落宫

- 用途：`star-placement-smoke.json`
- 盘面页截图：`/tmp/tiandiziwei-wave3-restart.png`
- 真太阳时参数串：`圻拆祗柝袛祇#2|2026|03|28|08|37|08|120.000000|-8|1|0`
- 北京时间参数串：`圻拆祗柝袛祇#1|2026|3|28|8|42|19|120.0|-8|1|0`
- 读取方式：
  - 通过 `ziweipan` 盘面截图人工核对。
  - 使用本机 Vision OCR 仅做放大辅助，最终以截图可见宫位文字和星曜排布为准。
- 计划写入的已确认值：
  - `辰` 宫主星：`武曲`、`七杀`
  - `卯` 宫主星：`天同`、`天梁`
  - `戌` 宫主星：`廉贞`、`贪狼`
- 额外人工确认（暂不写入 fixture，供快照级回归补强使用）：
  - `子` 宫主星：`巨门`
  - `丑` 宫主星：`天相`
  - `巳` 宫主星：`太阳`
  - `未` 宫主星：`紫微`、`破军`
  - `申` 宫主星：`天府`
  - `酉` 宫主星：`太阴`
- 读取备注：
  - `辰` 宫可见“武七 / 曲杀”成对排布，辅星另列，不混入正曜。
  - `卯` 宫可见“天天 / 同梁”成对排布，后续只编码正曜，不编码姚、廉、台、光、空等杂辅曜。
  - `戌` 宫可见“廉贪 / 贞狼”成对排布，旁侧的天月劫、魁德煞不纳入 `majorStarNames`。
  - 额外确认宫位均来自同一张 `/tmp/tiandiziwei-wave3-restart.png` 盘面截图，仍按“只认正曜、不把辅曜杂曜混入”原则读取。

## 样本 B: 四化

- 用途：`sihua-smoke.json`
- 盘面页截图：`/tmp/tiandiziwei-wave3-sample2.png`
- 真太阳时参数串：`圻拆祗柝袛祇#2|2026|04|01|10|07|15|120.000000|-8|1|0`
- 北京时间参数串：`圻拆祗柝袛祇#1|2026|4|1|10|11|13|120.0|-8|1|0`
- 读取方式：
  - 通过 `ziweipan` 盘面截图人工核对四化色块与所在星曜。
  - 使用本机 Vision OCR 辅助放大小字，字符串落盘时按 `ZiWeiChartComparableSnapshot` 现有协议写为 `星曜:化X`。
- 计划写入的已确认值：
  - `天同:化禄`
  - `天机:化权`
  - `文昌:化科`
  - `廉贞:化忌`
- 读取备注：
  - 底部中宫附近可见绿色“禄”标记，落在 `天同` 所在宫位。
  - 左上偏中宫位可见“权”标记，与 `天机` 所在宫位对应。
  - 左上宫位可见蓝色“科”标记，与 `文昌` 所在宫位对应。
  - 中部偏右宫位可见红色“忌”标记，与 `廉贞` 所在宫位对应。
  - 该样本最终数组顺序需按比较协议固定为：禄、权、科、忌。
